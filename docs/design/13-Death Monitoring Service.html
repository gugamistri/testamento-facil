BullMQ Job Scheduler

Sistema de filas baseado em Redis para agendamento de jobs recorrentes com retry logic e monitoramento via Bull Board.

// Daily cron job at 2 AM
queue.add('check-deaths', {}, {
repeat: { cron: '0 2 * * *' },
attempts: 3,
backoff: { type: 'exponential' }
});

Prevenção de Falsos Positivos

Sistema de Segurança Multi-Camada

Mecanismos rigorosos para evitar abertura indevida de testamentos

Verificação de Contato Ativa

Após detecção pela API, enviamos email e SMS com prazo de 48 horas para resposta. Cliente pode confirmar que está vivo com um clique.

Escalação de Contato:
D+0: Email + SMS inicial
D+1: Segundo email de lembrete
D+2: Terceiro email urgente
Link seguro com token HMAC de 7 dias

Exigência de Certidão de Óbito

Mesmo após validação em 3 camadas, advogado da plataforma só aprova abertura mediante upload de certidão oficial digitalizada.

Documentos Aceitos:
Certidão de óbito original (PDF)
Scan de cartório autenticado
Certidão digital com QR Code
Upload obrigatório antes de prosseguir

Cross-Verification de APIs

Informação de óbito deve ser confirmada por pelo menos 2 fontes independentes antes de avançar para revisão manual.

Fontes de Verificação:
Serasa API (Primária) Required
CRC Nacional (Primária) Required
SIM/SEADE (Backup) Optional
Mínimo 2 de 3 fontes devem concordar

Revisão Obrigatória por Advogado

Nenhum testamento é aberto sem aprovação final de advogado licenciado pela OAB após análise completa do caso.

Checklist de Revisão:
Verificação de identidade do falecido
Análise de certidão de óbito
Validação de dados cadastrais
Confirmação de herdeiros
Aprovação registrada com assinatura digital
